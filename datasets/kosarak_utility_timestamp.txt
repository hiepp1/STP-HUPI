    <h2 class="centered">Datasets </h2>
    <p>The SPMF software natively uses <strong>text files</strong> as input.
      Some small examples of text files that can be used with each algorithm are
      described in the <a href="https://www.philippe-fournier-viger.com/spmf/index.php?link=documentation.php">documentation</a>
      of SPMF. These sample input files can be downloaded from the download page
      (<a href="test_files.zip">test_files.zip)</a> for the release version of
      SPMF, and are included with the source code, for the source code version
      of SPMF. However, these datasets are quite small. For this reason, this
      webpage provides larger datasets that can be used with SPMF and that are
      often used in the data mining litterature for evaluating and comparing
      algorithm performance. Unless otherwise indicated, the datasets are in
      SPMF format.</p>
    <p>The <strong>datasets</strong> are divided in the following <strong>categories</strong>: </p>
    <ul>
      <li><span class="centered"><a href="#d1">Datasets for Sequential Pattern Mining / Sequential
      Rule Mining / Sequence Prediction</a></span>
        <ul>
          <li><a href="#r1">Real-life datasets in SPMF format</a></li>
          <li><a href="#r2">A collection of 30 books converted to SPMF format</a></li>
          <li><a href="#malware">Sequences of API calls of malware programs</a> </li>
          <li><a href="#mooc">Sequences of MOOC data with timestamps </a></li>
          <li><a href="#r3">Synthetic datasets</a></li>
          <li><a href="#timeintervals">Datasets of time-interval sequences</a> </li>
        </ul>
      </li>
      <li><a href="#d2">Datasets for Frequent Itemset
        mining / Association Rule Mining / Periodic pattern mining /
      Frequent episode mining)</a>
        <ul>
          <li><a href="#v1">Datasets in SPMF format</a></li>
          <li><a href="#v2">Synthetic datasets</a></li>
          <li><a href="#v3">Real-life datasets in SPMF format, having timestamps</a></li>
          <li><a href="#v4">          Real-life datasets in ARFF format</a></li>
        </ul>
      </li>
      <li><span class="centered"><a href="#d3">Datasets for </a></span><a href="#d3">High-Utility Pattern
      Mining</a>
        <ul>
          <li><a href="#c1">Real-life transaction datasets in SPMF format with real utility values </a></li>
          <li><a href="#c2"> Real-life transaction datasets in SPMF format having synthetic (fake) utility values </a></li>
          <li><a href="#c3">Datasets in SPMF format having utility values and timestamps</a></li>
          <li><a href="#c4">Datasets for high utility itemset mining with negative unit profit values</a></li>
          <li><a href="#c5">Datasets for high utility quantitative itemset mining</a></li>
          <li><a href="#c6">Datasets for on-shelf high-utility itemset mining with negative unit profit values</a></li>
          <li><a href="#c7">Datasets for high-utility sequential rule mining or high-utility sequential pattern mining</a><br>
          </li>
        </ul>
      </li>
      <li><a href="#d4">Datasets for  Cost-effective
      Pattern Mining (with utility and cost)</a></li>
      <li><a href="#d5">Graph datasets for graph pattern mining 
        </a>
        <ul>
          <li><span class="centered"><a href="#d6">Datasets for </a></span><a href="#d6"> mining subgraphs in
            a graph database </a></li>
          <li><span class="centered"><a href="#d7">Datasets for </a></span><a href="#d7"> mining subgraphs in
            a dynamic attributed graph</a></li>
        </ul>
      </li>
      <li><a href="#d8">Datasets for clustering</a></li>
    </ul>

        <h4><strong><a name="mooc" id="mooc"></a>Sequences of MOOC data with timestamps</strong> </h4>
        <p>This page also provides a time-extended sequence database that contains MOOC data (e-learning data) and can be used with sequential pattern mining algorithms such as SPM_FC_P and SPM_FC_L. It can be downloaded here: <a href="https://www.philippe-fournier-viger.com/spmf/datasets/mooc.txt">mooc.txt        </a>        </p>
        <p>This dataset was originally a  &ldquo;Course Recommendation&rdquo; dataset   provided by   the MoocData platform but has been transformed by Song et al. in SPMF format. The   original dataset was collected from XuetangX,   one of the largest   MOOC platforms in China. Originally used for course recommendation, the dataset contains the records of 82,535   course enrollment sequences from XuetangX from October   1, 2016 to March 31, 2018. The time span is 547&nbsp;days. The number of courses is 1,302. The number of sequences is 82,535. The length of the longest sequence is 398. The length of the shortest sequence is 3. The average sequence length is 5.19.</p>
        <p>See this paper for more details about this dataset and how it can be used: </p>
        <p><em> Song, W., Ye, W., Fournier-Viger, P. (2022). Mining sequential patterns with flexible constraints from MOOC data. Applied Intelligence</em> </p>

      <h4><strong><a name="v4"></a>Real-life datasets in ARFF format</strong></h4>
          <p>The GUI and command line interface of SPMF &gt; 093d and higher can
          read ARFF (Attribute Relational File Format) files . You can download
          a<a href="https://www.philippe-fournier-viger.com/spmf/uci_datasets.zip">
            collection of 36 real-life datasets in ARFF format from the UCI
            machine learning repository.</a> These files can be used with all
          association rule mining and itemset mining algorithms that take a
          transaction database as input. Note that examples <strong>MainTest.</strong>..in the
          source code have not been adapted to read ARFF files yet (for now,
          ARFF files are only accepted by the GUI and command line interface).
          Most features of the ARFF format are supported except that (1) the
          character = is forbidden and (2) escape characters are not
          considered. Note that when the ARFF format is used, the performance of
          the data mining algorithms will be slightly less than if the native
          SPMF file format is used because a conversion of the input file will
          be automatically performed before launching the algorithm and the
          result will have to be converted. This cost however should be small.
          The specification of the ARFF format can be found <a rel="nofollow" href="http://weka.wikispaces.com/ARFF+%28stable+version%29">here</a>.</p>

        <h3><strong><span class="centered"><a name="d4"></a>Datasets for </span> Cost-effective
        Pattern Mining </strong>(with utility and cost) </h3>
        <p>In the paper by Fournier-Viger et al. (2020), it was proposed to find <strong>cost-effective
          patterns</strong> in <strong>sequences</strong> <strong>with</strong>
        <strong>utility</strong> and <strong>cost</strong> <strong>information.
          </strong>This type of data is a set of sequences where each sequence
        is an ordered list of activity, where an activity has a cost value
        representing the amount of resources (e.g. time, money) spent to perform
        the activity. Moreover, each sequence has a utility label that can be
        either binary or numeric, which represents a positive or negative
        outcome. This type of data can represent for example how students use an
        e-learning system. In that case, each sequence is a list of activities
        or sessions performed by a student, and the cost represents the time
        spent for studying for each activity. On the other hand, the utility
        represents the outcome of a sequence such as the final exam score of a
        student after performing the acitivities (numeric utility), or whether
        he passed or failed the course or exam (binary utility). Another example
        of <strong>cost-utility sequences </strong>is hospital data, where a
        sequence is a list of medicines taken by some patient, the cost
        indicates how much money or time was spent for these medical treaments,
        and the utility is whether a patient has cured or died. </p>
      <p>From <strong>cost/utility sequences</strong>, we can then find
        patterns that have a <strong>low-cost </strong>but typically lead to a<strong>
          high utility (</strong>called<strong> low-cost high utility patterns</strong>
        or <strong>cost-efficient patterns</strong>) by applying algorithms
        such as <strong>CEPB, CEPN </strong>and <strong>CorCEPB</strong>. In
        the above examples, some cost-effective patterns may be that studying
        some e-learning materials A and B typically require not much time but
        leadw to high scores, or that taking some given medicines has a low cost
        but a high possibility of success to cure a disease. </p>

      <p>The <strong>two first datasets</strong> were obtained from the Web and
        are probably the<strong> two most famous datasets</strong> in <strong>subgraph
      mining</strong>. </p>
      <p>The <strong>other datasets </strong>were prepared by <strong>Dang
          Nguyen et al. </strong> based on data obtained from various data
        sources, and obtained from github.com/nphdang/gspan/, and used in this
        paper: Dang Nguyen, Wei Luo, Tu Dinh Nguyen, Svetha Venkatesh, Dinh
        Phung (2018). Learning Graph Representation via Frequent Subgraphs. SDM
        2018, San Diego, USA. SIAM, 306-314.). The descriptions of these latter
        datasets in the above table were obtained from that paper too. </p>
      <p>Note that some datasets have some label files. Those label files are
        not used by the algorithms offered in SPMF. They were generated when the
        datasets were prepared. These files indicates the correspondence between
        labels in the original datasets and those of the transformed datasets
        offered on this page. These label files are offered here because it can
        be useful to some people perhaps.</p>
    
    <h4><strong><span class="centered">Datasets for </span>Mining Patterns in
        Dynamic Attributed Graphs </strong><a name="d7"></a></h4>
      <p>A <strong>dynamic attributed graph</strong> is a graph that changes
        over time and where vertices may have multiple numerical attributes.
        Some algorithms are designed to discover interesting patterns in dynamic
        attributed graphs such as AER-Miner and TSEQMiner, offered in SPMF. The
        datasets used in the <a href="https://www.philippe-fournier-viger.com/2020_DAWAK_Attribute_Evolution_rules.pdf">AER-Miner</a>
        and <a href="https://www.philippe-fournier-viger.com/2019_KBS_Mining_significant_trend_sequences.pdf">TSEQMiner</a>
        papers are available here: <a href="datasets/DYNAMIC_GRAPH_DATASETS_REDUCED.zip">the
          datasets (ZIP, 5.0 MB) </a>and the <a href="datasets/DYNAMIC%20GRAPH%20DATASETS.zip">
          and some variations for scalability experiments (ZIP, 138 MB)</a>.
        Please see these papers for a description of these datasets. </p>
